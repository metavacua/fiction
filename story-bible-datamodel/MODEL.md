# SudoWrite Story Bible Data Model

This document defines a structured, local data model for SudoWrite's Story Bible. The purpose of this model is to allow for the programmatic creation, modification, and versioning of the core elements that guide SudoWrite's AI. By maintaining this data as code, we can integrate it into automated workflows, such as syncing with a canonical worldbuilding database like World Anvil.

The model is based on the official SudoWrite documentation and is designed to be represented in a clear, human-readable format like YAML.

## Top-Level Fields

### `braindump`
*   **Type:** String (Multiline)
*   **Purpose:** The initial, free-form container for all raw ideas about the story. This is the starting point for the entire creative process.
*   **Dependencies:**
    *   **Influences:** `synopsis`

### `genre`
*   **Type:** String
*   **Purpose:** Defines the genre of the story (e.g., "Hard Sci-Fi," "Cozy Mystery," "Epic Fantasy"). This sets a strong thematic and stylistic direction for the AI.
*   **Dependencies:**
    *   **Influences:** `synopsis`, `outline`, `scenes`, `draft`

### `style`
*   **Type:** String (Multiline)
*   **Purpose:** A detailed description of the narrative style and voice. This can be manually written or generated by SudoWrite's "Match My Style" feature from a sample text. It dictates the "feel" of the generated prose.
*   **Dependencies:**
    *   **Influences:** `scenes` (specifically the prose generation within them), `draft`

### `synopsis`
*   **Type:** String (Multiline)
*   **Purpose:** A concise summary of the story's plot. This is a critical field that provides the main narrative structure for the AI.
*   **Dependencies:**
    *   **Influenced by:** `braindump`, `genre`
    *   **Influences:** `characters`, `worldbuilding`, `outline`, `scenes`

### `characters`
*   **Type:** List of Objects
*   **Purpose:** Contains detailed profiles for the main characters. Each character is an object with its own set of attributes.
*   **Dependencies:**
    *   **Influenced by:** `synopsis`
    *   **Influences:** `outline`, `scenes`, `draft`
*   **Object Structure:**
    *   `name`: String (e.g., "Kaelen Voss")
    *   `description`: String (Detailed description of the character's personality, backstory, motivations, and appearance.)

### `worldbuilding`
*   **Type:** List of Objects
*   **Purpose:** Contains key details about the story's setting, magic systems, technology, locations, etc.
*   **Dependencies:**
    *   **Influenced by:** `synopsis`
    *   **Influences:** `outline`, `scenes`, `draft`
*   **Object Structure:**
    *   `name`: String (e.g., "The Neon Sprawl")
    *   `description`: String (Detailed description of the worldbuilding element.)

### `outline`
*   **Type:** List of Objects
*   **Purpose:** A chapter-by-chapter or scene-by-scene breakdown of the plot.
*   **Dependencies:**
    *   **Influenced by:** `genre`, `synopsis`, `characters`, `worldbuilding`
    *   **Influences:** `scenes`
*   **Object Structure:**
    *   `chapter_number`: Integer
    *   `title`: String (Optional title for the chapter)
    *   `summary`: String (A description of what happens in this chapter.)

### `scenes`
*   **Type:** List of Objects
*   **Purpose:** Detailed descriptions of individual scenes within the outline. This is the final step before generating full prose.
*   **Dependencies:**
    *   **Influenced by:** `genre`, `style`, `synopsis`, `outline`, `characters`, `worldbuilding`
    *   **Influences:** `draft`
*   **Object Structure:**
    *   `scene_number`: Integer
    *   `chapter_number`: Integer (To link it to the outline)
    *   `summary`: String (A detailed beat-by-beat description of the scene's events.)
    *   `prose`: String (The full, generated prose for the scene. This would be the output of a generation step.)
